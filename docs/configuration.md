# Configuration

This document covers every configuration surface in the HiClaw host-ops layer: environment variables, `openclaw.json` structure and managed fields, Traefik routing, nginx configs, MinIO storage layout, cron jobs, and state files.

---

## Configuration Surfaces

| Surface | Path | Who owns it |
|---|---|---|
| Runtime config | `workspace/openclaw.json` | Startup script, config keeper, gateway, reconciler |
| Manager startup | `start-manager-agent.sh` | This repo; copied into container by bootstrap keeper |
| Controller startup | `start-element-web.sh`, `start-tuwunel.sh` | This repo; copied into controller by bootstrap keeper |
| openclaw config watchdog | `manager-config-keeper.sh` | This repo; runs from host cron |
| Traefik routing | `traefik/claw.yml` (copy of `/data/coolify/proxy/dynamic/claw.yml`) | This repo; applied with `docker cp` |
| nginx inside controller | Generated by `start-element-web.sh` at container start | Generated; not a static file |
| oauth2-proxy credentials | `oauth2-proxy/.env` | Server-local ignored file; template is `oauth2-proxy/.env.example` |
| Allowed Google accounts | `oauth2-proxy/allowed-emails.txt` | This repo |

|---|---|---|---|

| `OPENCLAW_NO_RESPAWN` | manager | `1` | Forces openclaw to restart in-process on SIGUSR1 rather than re-exec. Required because Docker PID 1 has no supervisor to re-exec into. |

## openclaw.json Structure

Path on host: `/worksp/hiclaw/workspace/openclaw.json`
Path in manager container: `/root/manager-workspace/openclaw.json`
Path in controller container: `/root/hiclaw-fs/agents/manager/openclaw.json`

This file is the live runtime config for the OpenClaw gateway running inside `hiclaw-manager`. It has three concurrent writers:

- `start-manager-agent.sh` — rewrites key fields at every container start
- `manager-config-keeper.sh` — enforces schema invariants every minute from host cron
- The gateway's ManagerReconciler — writes its template to the file every ~47 seconds

OpenClaw also maintains a backup at `workspace/.openclaw/openclaw.json.bak` and a health baseline at `workspace/.openclaw/logs/config-health.json`. Both are used by the observe-recovery mechanism to revert "unexpected" changes — `manager-config-keeper.sh` updates both files atomically whenever it edits `openclaw.json`.

### `gateway`

```json
"gateway": {
  "auth": { "token": "<HICLAW_MANAGER_GATEWAY_KEY>" },
  "bind": "lan",
  "mode": "local",
  "port": 18799,
  "remote": { "token": "<HICLAW_MANAGER_GATEWAY_KEY>" },
  "controlUi": {
    "allowInsecureAuth": true,
    "allowedOrigins": ["*"],
    "dangerouslyDisableDeviceAuth": true
  }
}
```

`auth.token` and `remote.token` are both set to `HICLAW_MANAGER_GATEWAY_KEY` by the startup script. They are also injected verbatim into the nginx `manager-console.conf` sub_filter for auto-login token injection (see [nginx Upstream Resolution](#nginx-upstream-resolution)).

**WARNING — `controlUi` flags:**

| Flag | Current value | Why it is set | Safe to remove? |
|---|---|---|---|
| `allowInsecureAuth` | `true` | Allows the control UI to authenticate over plain HTTP without TLS. Required because the gateway runs on `http://` internally; TLS is terminated at Traefik. | Do not remove — removing it causes login failures when accessed via `control.claw.designflow.app`. |
| `dangerouslyDisableDeviceAuth` | `true` | Disables per-device auth enforcement in the control UI. Required because the auto-login mechanism injects a single shared token rather than issuing per-device credentials. | Do not remove unless you implement per-device token issuance. Removing it causes the injected token to fail device auth and the control UI becomes inaccessible. |
| `allowedOrigins` | `["*"]` | Allows the Control UI to be embedded from any origin. Required for Traefik's cross-origin proxy. | Can be tightened to `["https://control.claw.designflow.app"]` if cross-origin abuse is a concern — but test thoroughly before restricting. |

### `channels.matrix`

```json
"channels": {
  "matrix": {
    "accessToken": "<runtime value — set by startup script>",
    "autoJoin": "always",
    "blockStreaming": true,
    "dm": {
      "allowFrom": ["@manager:...", "@admin:..."],
      "policy": "allowlist"
    },
    "enabled": true,
    "encryption": false,
    "groupAllowFrom": ["@manager:...", "@admin:..."],
    "groupPolicy": "allowlist",
    "groups": {},
    "homeserver": "http://hiclaw-controller:6167",
    "network": { "dangerouslyAllowPrivateNetwork": true },
    "streaming": "partial",
    "userId": "@manager:matrix-local.hiclaw.io:18080"
  }
}
```

`accessToken` and `homeserver` are overwritten by the startup script on every start using the value obtained from `POST /login` against the Tuwunel homeserver. Do not hand-edit these — they are valid for the current Matrix session only.

`channels.matrix.groups` must never contain a wildcard key `"*"`. The OpenClaw schema rejects additional properties in the groups object, causing every config reload and `update.run` to fail with `managed-service-handoff-unavailable`. The config keeper strips this key on every run.

`network.dangerouslyAllowPrivateNetwork` is required because the Matrix homeserver is on a private Docker network. Without it the gateway refuses to connect to non-routable IPs.

### `session`

```json
"session": {
  "dmScope": "main",
  "resetByType": {
    "dm": { "atHour": 4, "mode": "daily" },
    "group": { "atHour": 4, "mode": "daily" }
  }
}
```

`session.dmScope = "main"` forces all Matrix DMs onto the agent's primary session key (`agent:main:main`), so the HiClaw chat UI and OpenClaw web chat share one transcript. The config keeper enforces this value. Do not change it to `"per-dm"` — doing so creates isolated session contexts per DM thread and the two UIs diverge.

`resetByType` schedules daily session resets at 04:00 UTC for both DM and group sessions.

### `commands`

**This is the most non-obvious part of the system. Read this before touching anything related to `commands`.**

The gateway uses a startup-time baseline for reload diffs, not its in-memory running state. In the current stable path, `start-manager-agent.sh` removes `commands.restart` before launching the gateway, and `manager-config-keeper.sh` removes it again if it appears later. The stable host file currently has no `commands` key.

If `commands` changes in a way OpenClaw treats as a restart signal, the gateway triggers a full in-process restart (SIGUSR1) instead of a hot reload. A persistent `commands.restart` was a previous restart-loop source.

| File state | Result |
|---|---|
| No `commands` key | Current stable state |
| `commands: null` | Reconciler drift; keeper may leave/normalize depending on full config state |
| `{"restart": true}` | Must be cleared by startup/keeper |
| `{"restart": false}` | Invalid for this deployment; clear it |

**Do not persist `commands.restart`.** A broken state produces a restart loop visible in `docker logs hiclaw-manager` as repeated restart/reload entries.

To inspect the startup baseline:

```bash
sudo python3 -c "
import json
h = json.load(open('/worksp/hiclaw/workspace/.openclaw/logs/config-health.json'))
for path, info in h['entries'].items():
    lg = info['lastKnownGood']
    print(path, '  hash:', lg['hash'][:16], ' bytes:', lg['bytes'])
"
```

To verify the keeper is writing the correct value:

```bash
sudo python3 -c "import json; d=json.load(open('/worksp/hiclaw/workspace/openclaw.json')); print('commands:', d.get('commands'))"
# Expected: None
```

### `agents`

```json
"agents": {
  "defaults": {
    "bootstrapMaxChars": 20000,
    "maxConcurrent": 4,
    "model": { "primary": "hiclaw-gateway/deepseek/deepseek-v4-pro" },
    "models": {
      "hiclaw-gateway/deepseek/deepseek-v4-pro": { "alias": "deepseek/deepseek-v4-pro" },
      ... (one entry per model above)
    },
    "memorySearch": {
      "model": "text-embedding-v4",
      "provider": "openai",
      "remote": {
        "apiKey": "<HICLAW_MANAGER_GATEWAY_KEY>",
        "baseUrl": "http://aigw-local.hiclaw.io:8080/v1"
      }
    },
    "subagents": { "maxConcurrent": 8 },
    "timeoutSeconds": 1800,
    "workspace": "~"
  }
}
```

`bootstrapMaxChars: 20000` is enforced by the config keeper. The default OpenClaw value (8000) truncates AGENTS.md (15 KB+). Do not reduce this.

`agents.defaults.models` is rebuilt by the startup script from the full model list. Each entry maps the qualified ID (`hiclaw-gateway/<model-id>`) to a short `alias` used in the UI dropdown.

`memorySearch` is only present when `HICLAW_EMBEDDING_MODEL` is set. The remote block uses the same gateway key and AI gateway URL as `models.providers.hiclaw-gateway`.

## Keeper-Managed Fields

`manager-config-keeper.sh` runs every minute from host cron and enforces these invariants:

| Field | Enforced value | Why |
|---|---|---|
| `commands.restart` | Removed if present | Prevents restart-trigger field from persisting in live config |
| `session.dmScope` | `"main"` | Keeps HiClaw chat and OpenClaw web chat on one shared session |
| `channels.matrix.groups["*"]` | Removed if present | OpenClaw schema rejects wildcard key; its presence breaks all config reloads |
| `channels.matrix.groups[*].allow` | Renamed to `enabled` | Controller writes `allow`; schema requires `enabled` |
| `plugins.load.paths` | Clawtalk npm path removed | ClawTalk loads from the host-backed bundled shim, not the npm path |
| `plugins.entries.clawtalk` | Added if absent | Ensures ClawTalk is always registered |
| `agents.defaults.bootstrapMaxChars` | `20000` | Prevents AGENTS.md truncation at the default 8000-char limit |
| `models.providers.hiclaw-gateway.models[*].contextWindow` | Static override table for current alias IDs plus `deepseek/deepseek-v4-pro` | Reconciler restores stale values from its internal state; OpenRouter sync only updates canonical IDs when available |

When the keeper makes changes, it also:
1. Deletes `workspace/openclaw.json.bak` — prevents observe-recovery from reverting the fix
2. Updates `config-health.json` with a fresh hash, byte count, and file metadata — prevents observe-recovery from flagging the keeper's version as "unexpected" and reverting it

## Startup Script Dynamic Fields

`start-manager-agent.sh` overwrites these fields on every container start:

| Field | Source |
|---|---|
| `channels.matrix.accessToken` | Fresh Matrix login token from `POST /login` to Tuwunel |
| `channels.matrix.homeserver` | `HICLAW_MATRIX_URL` env var |
| `channels.matrix.userId` | Constructed from `HICLAW_MATRIX_DOMAIN` |
| `models.providers.hiclaw-gateway.apiKey` | `HICLAW_MANAGER_GATEWAY_KEY` |
| `models.providers.hiclaw-gateway.baseUrl` | `HICLAW_AI_GATEWAY_URL` |
| `models[*].contextWindow` and `models[*].maxTokens` | OpenRouter API (for matching models only) |
| `gateway.auth.token`, `gateway.remote.token` | `HICLAW_MANAGER_GATEWAY_KEY` |
| `agents.defaults.model.primary` | `HICLAW_DEFAULT_MODEL` |
| `agents.defaults.models` | Rebuilt from full model list |
| `agents.defaults.memorySearch` | `HICLAW_EMBEDDING_MODEL` and `HICLAW_AI_GATEWAY_URL` |
| `commands.restart` | Removed with `del(.commands.restart)` before launch; keeper clears it if it reappears |

After the OpenRouter sync, the startup script immediately pushes the updated `openclaw.json` to MinIO so the background MinIO-to-Local sync loop (which starts a few seconds later) does not overwrite the fresh values with stale MinIO data.

## openclaw Install Paths

| Path | When present |
|---|---|
| `/opt/openclaw/` | Always — bundled in the base image |
| `/usr/lib/node_modules/openclaw/` | After a successful `openclaw update` via the keeper |

The startup script validates the npm install before symlinking it. Both `node_modules/json5/package.json` and `node_modules/openai/index.mjs` must exist. A partial OOM-interrupted npm install leaves `openclaw.mjs` present but all `.mjs` files absent from the `openai` package — `node openclaw.mjs --version` succeeds but the gateway crashes on first use. If either check fails, the broken install is removed and the symlink falls back to the base image version.

The package hash is written to `workspace/.openclaw-startup-pkg-hash` on every start. `manager-bootstrap-keeper.sh` polls this file and restarts the container when the hash changes (indicating a successful `openclaw update` completed).

## openclaw Update Mechanism

Triggered by clicking "Update now" in the Control UI at `control.claw.designflow.app`.

1. openclaw's `update.run` invokes `systemd-run --user --scope` to spawn a detached update helper.
2. The startup script installs a fake `/usr/local/bin/systemd-run` that writes `/root/manager-workspace/.openclaw-update-requested` (visible on the host at `workspace/.openclaw-update-requested`) and exits 0.
3. openclaw believes the handoff started successfully and sends itself SIGUSR1 (in-process restart). This is expected — it kills the active agent session.
4. `manager-bootstrap-keeper.sh` (cron, every minute) detects the marker file, removes it, then runs: `docker exec hiclaw-manager openclaw update --yes --json`
5. The keeper sleeps 30 seconds after the update completes to allow openclaw's in-process SIGUSR1 restart (which rewrites `openclaw.json`) to finish before `docker restart` is called. **This sleep is critical** — without it, `docker restart` kills the container mid-write and produces a truncated `openclaw.json` (the 8788-byte truncation incident, 2026-05-31).
6. The keeper detects the changed package hash and calls `docker restart hiclaw-manager` to load the new binary.

Do not run `openclaw update` directly inside the container. The keeper handles memory limits (`--memory-swap 3g` via `docker update`) that are required to prevent npm OOM mid-install.

## nginx Configs (inside hiclaw-controller)

Generated by `start-element-web.sh` at every container start. Not static files — changes require editing `start-element-web.sh` and restarting the controller.

### `manager-console.conf` (port 18888)

Proxies `control.claw.designflow.app` to the manager's OpenClaw gateway at `hiclaw-manager:18799`. Key behaviors:

- **Docker DNS resolver** (`127.0.0.11 valid=10s`): The upstream hostname is stored in a `$upstream` variable rather than used directly. This forces nginx to re-resolve `hiclaw-manager` on each request, so IP changes after `docker restart hiclaw-manager` are transparent. Do not replace with a hardcoded IP — doing so causes 502 errors after every manager container recreation.
- **Auto-login token injection**: A `sub_filter` injects an inline `<script>` that sets `location.hash` to `#token=<HICLAW_MANAGER_GATEWAY_KEY>` on page load. This is required because the control UI reads the gateway token from the URL hash, not from a cookie or stored credential.
- **CSP stripping**: `proxy_hide_header Content-Security-Policy` allows the injected inline script to execute.
- **WebSocket support**: `proxy_set_header Upgrade`, `Connection "upgrade"`, and extended timeouts (3600s) support the gateway's real-time WebSocket connections.

### `element-web.conf` (port 8088)

Serves the Element Web SPA. Key behaviors:

- **JS injection**: `sub_filter '</head>'` appends five scripts: `browser-bypass.js` (sets `mx_accepts_unsupported_browser`), `auto-login.js` (injects Matrix session into localStorage), `auth-ui-tweaks.js` (styles SSO button, auto-clicks skip-verification), `control-panel-btn.js` (fixed "Control Panel" button), `new-chat-btn.js` ("+ New Chat" button).
- **hiclaw-chat-api.py proxy**: Locations `/hiclaw-api/new-chat`, `/hiclaw-api/session`, `/hiclaw-api/matrix-auth`, `/hiclaw-api/healthz` proxy to the Python API server at `127.0.0.1:8091`.
- **config.json**: Served with `no-cache`; a `sub_filter` replaces `__PUBLIC_BASE_URL__` with the current request scheme and host, so the homeserver URL adapts correctly when accessed from different domains.

### `plugin-server.conf` (port 8002)

Serves Higress WASM plugin binaries from `/usr/share/nginx/html/plugins/` to Envoy's filter-chain loader. Required for AI plugins (LLM request rewriting) to function. If this server is down, Envoy cannot fetch plugin bytecode and all AI gateway requests fail.

## `config-health.json`

Path: `workspace/.openclaw/logs/config-health.json`

This file is OpenClaw's config-health baseline. It records the hash, byte count, and filesystem metadata of `openclaw.json` at the time the gateway last processed it as "last known good". The observe-recovery mechanism uses this baseline to detect and revert "unexpected" changes.

`manager-config-keeper.sh` updates this file atomically whenever it writes `openclaw.json`. If `config-health.json` is not updated to match a keeper-written `openclaw.json`, observe-recovery will flag the keeper's version as unexpected and revert it within minutes.

`start-manager-agent.sh` deletes `config-health.json` entirely at startup so the gateway establishes a fresh baseline from the startup-patched `openclaw.json`. This ensures the startup script's changes (Matrix token, model metadata, and command cleanup) become the new baseline rather than being reverted.

## Known Configuration Smells

- **ClawTalk API key is hardcoded** in `manager-config-keeper.sh` (`cc_live_d5a5025bc0dc6894ac8acc6f867b336667e3e104`). It belongs in an env var or secret store. Accepted for a single-deployment host-ops layer.
- **`fix-element-config.sh` hardcodes the gateway key** in the nginx `manager-console.conf` sub_filter for auto-login. The key is already in the manager container's environment — the hardcoded value is a convenience for the one-off repair script only.
- **OpenRouter model sync is partial.** Startup attempts to map gateway aliases to canonical OpenRouter IDs before syncing metadata, but the live config can still contain alias IDs after reconciler drift. `manager-config-keeper.sh` enforces static context-window values for all current aliases.
- **`gateway.controlUi.dangerouslyDisableDeviceAuth = true`** and **`allowInsecureAuth = true`** are required for the auto-login token injection to work. These flags are acceptable on a server gated by Google OAuth at the Traefik layer, but they should not be present in a multi-user deployment.
- **`oauth2-proxy/.env` contains real credentials on the server** and is ignored by git. Rotate credentials by updating this file and restarting the container.
